## 1. Who Should Use This Guide
This guide is for first-time users, HR teams, interviewers, and administrators who want to start using HireOps quickly and correctly.

## 3. First Login Steps
1. Open the frontend application URL.
2. Sign in using the provided credentials.
3. Verify that your dashboard opens successfully.
4. Confirm your role-based navigation menu is visible.

## 4. Typical Hiring Flow In HireOps
1. Create a job posting with role details and requirements.
2. Upload candidate resumes for the selected job.
3. Let AI parsing and matching process candidate profiles.
4. Review candidate list and shortlist with score context.
5. Create interview rounds and assign evaluators.
6. Schedule interviews and collect feedback/results.
7. Progress selected candidates to final stage and hiring decision.

## 5. Recommended Daily Operating Pattern
- Start from Dashboard for hiring pipeline overview.
- Monitor candidate progression and blocked rounds.
- Review interview requests and pending decisions.
- Close the day by checking logs and outcomes.

## 6. Role Responsibilities

### HR
- Manage jobs, candidate uploads, shortlisting, rounds, and final decisions.

### Interviewer
- Review assigned interviews, provide feedback, and submit evaluations.

### Admin
- Manage users, roles, and overall system governance.

## 7. Troubleshooting Basics
- If login fails, verify frontend API environment variable configuration.
- If resume processing fails, check backend and server health endpoints.
- If pages break in production, confirm all service URLs are set in env vars.
